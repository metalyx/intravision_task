Разработал SPA на react, использовал state-manager - redux.
Небольшой роутинг с помощью react-router-dom. 
Для отладки - redux-logger. 
Для асинхронных запросов к api - redux-thunk.

Чтобы запустить проект:
1) Распакуйте архив в одну папку. 
2) В папке проекта npm install в терминале или PowerShell.
3) В папке проекта npm start в терминале или PowerShell.

Приложение будет доступно локально по адресу http://localhost:3000/intravision_task

Если нужен production build - то в терминале, в папке проекта npm build. 

Описание что где лежит:

- В папке shared лежат картинки для бокового меню, шрифты и api.js, который содержит в себе константы для запросов к api.
Если необходимо сменить tenantguid - меняйте константу "tenantguid" в файле api.js.

В папке redux распологается три файла, в actions.js все action'ы, в reducers.js все reducer'ы, в configureStore с помощью "combineReducers" объединяем все reducer'ы.
А также подлкючаем thunk и logger.

В папке css распологаются стили. 

В папке components - react-компоненты.

- MainComponent.js - главный компонент, здесь мы делаем запросы к api при запуске приложения. Собираем из redux store все, что нужно - передаем в menu.
- MenuComponent.js - компонент, отвечающий за роутинг и за меню. Также здесь рендерятся заглушки с названием вкладок, которые реализовывать не нужно.
- SearchComponent.js - компонент, отвечающий за отображение поиска сверху.
- ContentComponent.js - отвечает за отображение всех заявок и вызова RequestComponent.js с соответствующими props.
- RequestComponent.js - отвечает за отображение либо модуля с созданием новой заявки, либо с редактированием выбранной пользователем заявки.
  В нем же форматируются все данные из api, для корректного вывода пользователю.

- Обновление через вебсокеты не делал, ограничился window.location.reload() при сохранении изменений в заявке.
- UI-библиотек не подключал, писал свой CSS.
- Валидации форм как таковой нет, если пользователь не менял ничего в форме редактирования - запрос к api не происходит. Если что-то поменял(написал комментарий) данные отправляются в api.
  Аналогично с созданием новой заявки - если название заявки пустое - заявка не создается.
